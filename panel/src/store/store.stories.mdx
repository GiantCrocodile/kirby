# `$store` - shared data and state management

We use [Vuex](https://vuex.vuejs.org/) to ensure a consistent state of data throughout the Panel.  
If you have not worked with Vuex before, you should first familiarize yourself with [their documentation](https://vuex.vuejs.org/guide/).

```js
this.$store
```

## Core modules
- `content`
- `heartbeat`
- `languages`
- `notification`
- `system`
- `translation`
- `user`

## Adding your own Vuex module

Using the `created(app)` callback, you can register your own [Vuex module](https://vuex.vuejs.org/guide/modules.html) in your plguin's JavaScript definition:

```js
// Create your own Vuex module

const yourModule = {
  namespaced: true,
  state: {
    // …
  },
  getters: {
    // …
  },
  mutations: {
    // …
  },
  actions: {
    // …
};


// Register it with the Panel app

panel.plugin("your/plugin", {
  created(app) {
    app.$store.registerModule("yourModule", yourModule);
  }
});

```
